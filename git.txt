ADD
git add . 
- แอดทุกไฟล์ที่เป็น Untracking (U)
git add rpa.js
- แอดไฟล์ชื่อ rpa.js

COMMIT
git commit -am "note"
- auto commit ไฟล์ทั้งหมดที่เคยแอดแล้ว
git commit -m "note"
- commit เฉพาะไฟล์ที่ add แล้ว

LOG
git log
- log ที่เคยแอดไปทั้งหมดจะขึ้นมาแบบละเอียด
git log --oneline
- log ที่เคยแอดไปทั้งหมดจะขึ้นมาให้ดูง่าย เนื่องจากโชว์ไม่เยอะ

STATUS
git status
- ตรวจสอบสถานะ git ว่ามีตัวไหน add , commit , untracking บ้าง

RESET (ไม่ควรใช้ถ้าแชร์โปรเจค)
git reset --hard HEAD
- Reset ทุกอย่างที่ Add เข้า git ทำล่าสุด (*HEAD คือที่ git commit ล่าสุด) (ไม่ทำ Untracking File)
git clean -df (ไม่ต้องใส่ path ที่จะคลีน คลีนทั้งหมด)
- clear ทุกอย่างที่ untrack อยู่
git reset --hard xxxx (รหัสgitเช็คจาก status (เรียกกลับมาก็ได้แค่รู้รหัส))
- จะเป็นการเลือกความทรงจำที่แอดไว้ว่าให้กลับไปอยู่ช่วงไหน

REVERT
git revert --no-edit HEAD
- Revert เอาตัวที่ commit ก่อนหน้าตัวล่าสุดขึ้นมาใช้ (ไม่ได้ถูกลบทิ้ง แค่ ignore)
git revert --no-edit --no-commit xxxx..HEAD (ex. รหัส git..HEAD)
- เป็นการย้อนหลายความทรงจำ จะเริ่มต้นจากช่วงของรหัส แต่จะละทิ้งตัวที่ต่อจากรหัส ถึง HEAD *แต่ต้องมา commit ใหม่
git revert --no-edit --no-commit xxxx^..HEAD (ex. รหัส git^..HEAD)
- จะเริ่มต้นย้อนจากช่วงรหัสนั้น ถึง HEAD ที่จะถูกลบความทรงจำออก ให้กลับมาที่ที่เราต้องการ *แต่ต้องมา commit ใหม่

REMOTE
git remote add origin xxxx (https://github.com/warudd/api_rpa.git แล้วแต่ respository)
- เป็นการ remote เพื่อ add git เข้าสู่ github
git remote
- เช็คว่า add origin เสร็จสิ้นหรือยัง
*origin คือ ผู้ให้บริการเช่น github ให้บริการที่เก็บโค้ด ก็คือลิ้งในวงเล็บนั่นแหละ
git remote -v
- check origin link ที่เชื่อมต่ออยู่
git remite rm xxxx (ชื่อด้านหน้าเช่น origin)
- จะทำการเอา origin ที่เชื่อมต่อกับลิ้งของ github ที่ไม่ต้องการออก

BRANCH
git branch
- เช็คสาย branch ว่ามีกี่สายควรใช้ชื่อ main ใน branch หลัก
git branch -M main
-เปลี่ยนชื่อ branch เป็น "main"

PUSH
git push origin main 
- อัพ branch main ขึ้น github
git push -u origin main
ถ้าไม่ใส่ -u คือมันจะไม่จดจำว่าเราจะ push ไปที่ github ในครั้งหน้า หรือเวลาจะ pull ถ้า push -u แล้วเราไม่ต้องระบุ branch main ในการ pull
**แต่ครั้งถัดไปไม่ต้องใส่ -u แล้ว

PULL
(หมายถึง pull มาอัพเดทที่เครื่องเรา)
git pull 
ถ้ามีอัพเดทบน github เพิ่มแต่ local เราไม่มี จะ pull มาอัพเดทบนคอมเรา

CLONE
